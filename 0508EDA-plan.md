## 0. 수집한 상품 페이지 feature 품질 확인

### 목적

모델 학습 전에 상품 페이지 데이터가 충분히 깨끗하고 일관적인지 확인한다.

### 확인할 항목

| 항목 | 확인 내용 |
| --- | --- |
| 결측치 | 상품명, 가격, 용량, 리뷰 수, 평점, 상세 설명 등 주요 feature 누락 여부 |
| 이상치 | 지나치게 낮거나 높은 가격, 비정상적인 용량, 과도하게 긴/짧은 설명 |
| 중복 상품 | 동일 상품이 여러 번 수집되었는지 |
| 텍스트 품질 | 상세 설명이 너무 짧거나 의미 없는 문구로 채워져 있는지 |
| 단위 통일 | 가격, 용량, ml당 가격 등의 단위가 일관적인지 |

### 주요 산출물

- Feature별 결측률 표
- 가격/용량/ml당 가격 이상치 리스트
- 중복 상품 리스트
- 텍스트 길이 분포
- 이미지 누락률

## 1. 수집한 상품 페이지들의 시장 대표성 확인

### 목적

현재 수집한 **300개 폼클렌저 상품 pool이 실제 시장/경쟁 환경을 충분히 대표하는지** 확인한다.

모델 결과는 “폼클렌저 300개 pool 기준”으로 해석되므로, 이 pool이 얼마나 대표적인지가 중요하다.

### 확인할 항목

| 항목 | 확인 내용 |
| --- | --- |
| 브랜드 분포 | 특정 브랜드에 과도하게 치우쳐 있지 않은지 |
| 가격대 분포 | 저가/중가/고가 상품이 골고루 포함되어 있는지 |
| 제품 타입 | 폼, 젤, 크림, 약산성, 저자극 등 유형 분포 |
| 피부 타입 | 지성, 건성, 민감성, 여드름성 등 타겟 피부 타입 분포 |
| 용량 분포 | 너무 특정 용량대에 몰려 있지 않은지 |
| ml당 가격 분포 | 실질 가격 경쟁력 분포 확인 |
| 리뷰/평점 분포 | 인기 상품과 롱테일 상품이 함께 포함되어 있는지 |

### 주요 산출물

- 브랜드별 상품 수 bar chart
- 가격대 histogram
- 제품 타입별 분포
- 피부 타입별 분포
- 용량 분포
- ml당 가격 분포
- 리뷰 수/평점 분포

### 해석 포인트

> 300개 pool이 주요 브랜드, 가격대, 제품 타입, 피부 타입을 고르게 포함한다면, 해당 pool 내 ranking과 추천 확률은 “폼클렌저 시장 내 AI 추천 가능성”으로 의미 있게 해석할 수 있다.

## 2. AI 응답 데이터 기반 후보 set 구성 품질 확인

### 목적

AI에게 제시한 3개 후보 조합이 균형 있게 구성되었는지 확인한다.

모델은 3개 후보 ranking 데이터를 기반으로 학습하므로, 후보 set이 편향되어 있으면 recommendation score도 왜곡될 수 있다.

### 확인할 항목

| 항목 | 확인 내용 |
| --- | --- |
| **상품별 등장 횟수** | **특정 상품이 너무 많이/적게 등장하지 않았는지** |
| **같은 상품 조합 반복 횟수** | **동일한 3개 조합이 과도하게 반복되지 않았는지** |
| **상품 pair 반복 횟수** | **같은 두 상품이 너무 자주 같이 등장하지 않았는지** |
| **브랜드 조합** | 같은 브랜드끼리만 비교되거나 특정 브랜드 조합이 과도하지 않은지 |
| **가격대 조합** | 저가끼리만, 고가끼리만 비교되는 등 편향이 없는지 |
| **제품 타입 조합** | 같은 제품 타입끼리만 비교되지 않았는지 |
| **Comparison network 연결성** | 전체 상품 pool이 비교 네트워크상 하나로 연결되는지 |

### Comparison network란?

상품을 node, 함께 비교된 관계를 edge로 보는 방식이다.

예:

```
A, B, C가 한 후보 set에 함께 등장
→ A-B, A-C, B-C가 비교 관계로 연결됨
```

이 **네트워크가 잘 연결**되어 있어야 전체 300개 상품의 score를 같은 기준 위에서 비교할 수 있다.

### 주요 산출물

- 상품별 등장 횟수 분포
- 상품 pair별 동시 등장 횟수 분포
- 반복 후보 set 리스트
- 브랜드/가격대별 후보 조합 분포
- comparison network 연결성 요약
- 네트워크에서 고립된 상품 또는 약하게 연결된 상품 리스트

### 해석 포인트

> 모든 상품이 다양한 상대와 충분히 비교되고, comparison network가 하나로 연결되어 있다면 모델이 300개 상품을 같은 scale에서 ranking할 수 있다.
>

## 3. AI 응답 데이터 결과 분석

### 목적

AI가 실제로 어떤 상품을 선호했는지, 그리고 응답에 편향이 있는지 확인한다.

### 확인할 항목

| 항목 | 확인 내용 |
| --- | --- |
| **상품별 1위 횟수** | 어떤 상품이 가장 자주 top pick 되었는지 |
| **상품별 평균 ranking** | 전반적으로 어떤 상품이 상위권에 위치하는지 |
| **상품별 win rate** | 다른 상품 대비 얼마나 자주 이겼는지 |
| **rank별 feature 차이** | 1위 상품과 3위 상품의 feature가 어떻게 다른지 |
| **브랜드별 평균 rank** | 특정 브랜드가 AI에게 더 선호되는지 |
| **가격대별 평균 rank** | 가격대에 따른 선호 차이가 있는지 |
| **제품 타입별 평균 rank** | 약산성, 저자극 등 제품 타입별 차이 |
| **Position bias** | 후보 제시 순서가 AI ranking에 영향을 주는지 |

### Position bias 확인 예시

AI에게 후보가 다음 순서로 제시되었다고 할 때:

```
item1, item2, item3
```

각 위치가 1위로 선택된 비율을 확인한다.

| 제시 위치 | 1위 선택 비율 |
| --- | --- |
| item1 | 45% |
| item2 | 30% |
| item3 | 25% |

만약 item1이 과도하게 많이 1위로 선택된다면, 실제 상품 선호가 아니라 제시 순서에 따른 편향이 있을 수 있다.

### 주요 산출물

- 상품별 1위 횟수 ranking
- 상품별 평균 rank table
- 상품별 win rate table
- rank별 평균 가격/리뷰/평점/ml당 가격 비교
- 브랜드별 평균 rank
- 제품 타입별 평균 rank
- 후보 제시 위치별 1위 선택 비율

### 해석 포인트

> AI ranking 결과에서 특정 feature나 브랜드, 가격대가 반복적으로 상위 rank와 관련되는지 확인하면 모델이 학습할 수 있는 signal이 있는지 사전에 판단할 수 있다.
>

## 4. 간단한 baseline 지표 확인

### 목적

복잡한 모델을 학습하기 전에, 단순 지표만으로도 AI 선호 패턴이 보이는지 확인한다.

이 단계는 모델 성능을 비교할 기준선을 만드는 역할을 한다.

### 확인할 baseline 지표

| 지표 | 의미 |
| --- | --- |
| Top1 rate | 해당 상품이 등장했을 때 1위로 선택된 비율 |
| 평균 순위 | 해당 상품이 등장했을 때 평균적으로 몇 위였는지 |
| Win rate | 비교 가능한 상대들에 대해 이긴 비율 |
| Borda score | 1위=3점, 2위=2점, 3위=1점으로 환산한 점수 |
| 브랜드별 평균 rank | 브랜드 단위의 단순 선호도 |
| 가격대별 평균 rank | 가격대별 단순 선호도 |
| 제품 타입별 평균 rank | 제품 타입별 단순 선호도 |

### 예시

| 상품 | 등장 횟수 | 1위 횟수 | Top1 rate | 평균 순위 | Win rate |
| --- | --- | --- | --- | --- | --- |
| A | 10 | 5 | 50% | 1.6 | 75% |
| B | 10 | 2 | 20% | 2.3 | 40% |
| C | 10 | 1 | 10% | 2.6 | 25% |

### 주요 산출물

- 상품별 Top1 rate table
- 상품별 평균 순위 table
- 상품별 win rate table
- Borda score ranking
- baseline 기준 상위/하위 상품 리스트

### 해석 포인트

> baseline 지표에서 이미 일관된 선호 패턴이 보인다면, ranking 모델이 학습할 수 있는 signal이 존재한다고 볼 수 있다.
>

## 한 줄 요약

EDA에서는 단순히 상품 데이터의 분포만 보는 것이 아니라, **상품 pool이 대표성 있는지, AI 비교 실험이 균형 있게 설계되었는지, AI ranking 결과에 학습 가능한 signal이 있는지**를 확인하는 것이 핵심이다.
