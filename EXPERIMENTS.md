# 실험 버전 정리

AI가 폼클렌저 상품을 순위 매기는 방식을 예측하는 모델을 단계적으로 개선한 기록입니다.  
각 버전이 **무엇을 바꿨는지**, **왜 바꿨는지**, **어떤 결과가 나왔는지**를 정리합니다.

---

## 공통 설정 (모든 버전에서 동일)

| 항목 | 내용 |
|------|------|
| 학습 방식 | 3개 상품 중 AI가 매긴 순위를 학습 목표로 사용 (Plackett-Luce 방식) |
| 평가 기준 | 새로운 브랜드(학습 중 한 번도 본 적 없는 상품)에 대한 1위 예측 정확도 |
| 상품 수 | 폼클렌저 300개 |
| 비교 세트 수 | 5,298개 (3개 상품씩 묶인 비교 실험) |
| 학습/검증/테스트 비율 | 60% / 11% / 29% (테스트는 11개 브랜드 완전 분리) |

---

## 개선 방향 1: 학습 데이터 정제

### AI 엔진 분리 학습

**문제:** OpenAI와 Anthropic 두 AI 엔진이 같은 상품 세트에 서로 다른 순위를 매기는 경우가 많습니다.  
두 엔진의 레이블을 섞어 학습하면 모델이 일관성 없는 신호를 받아 성능이 떨어집니다.

**해결:** 한 번에 하나의 AI 엔진 데이터만 사용해 학습합니다.  
현재 기본 설정은 OpenAI 데이터만 사용합니다.

**효과:** 학습 데이터가 절반으로 줄지만(3,594개 → 1,797개 비교 세트), 레이블 일관성이 높아져 학습이 안정됩니다.

---

## 개선 방향 2: 입력 특징 구성

### 버전 1 (기준선) — 특징 24개

처음 수집한 상품 페이지 수치 24가지를 그대로 사용합니다.

| 분류 | 포함된 특징 |
|------|------------|
| 텍스트 구성 | 전체 텍스트 길이, 이미지 수, 표 수, 목록 항목 수, 단락 수, 섹션 수 |
| 신뢰도 표현 | 구조화 데이터 항목 수, 구체적 숫자 표현 수, 모호한 표현 수, 수치 구체성 비율 |
| 가격 / 용량 | 가격(원), 용량(ml) |
| 성분 / 효능 | 주요 성분 수, pH 수치, 효능 키워드 수, 텍스처 키워드 수, 부정적 표현 수 |
| 피부 타입 | 대상 피부 타입 수 |
| 리뷰 | 평점, 리뷰 수 |
| 인증 | 인증 마크 수 |
| 소셜 증거 | 소셜 증거 요소 수, 외부 기관 인용 수 |
| 종합 점수 | AI 친화도 종합 점수 |

---

### 버전 2 — 특징 20개 (결측값 많은 항목 제거)

**문제:** 일부 특징은 데이터가 너무 많이 빠져 있어 오히려 모델에 노이즈가 됩니다.

| 제거된 특징 | 데이터 누락 비율 |
|------------|----------------|
| pH 수치 | 98.2% 누락 |
| 평점 | 38.8% 누락 |
| 리뷰 수 | 38.5% 누락 |
| 용량(ml) | 19.9% 누락 |

**해결:** 누락 비율이 20%를 넘는 특징 4가지를 제거합니다.  
나머지 20개 특징만 사용합니다.

---

### 버전 3 — 특징 20개 + 로그 변환 + 제시 순서 추가

**문제 1: 수치 단위 왜곡**  
가격은 수만 원 단위이고, 이미지 수는 수십 개 단위입니다.  
이런 수치를 그대로 쓰면 큰 숫자가 모델에 과도하게 영향을 미칩니다.  
또한 "이미지 10개와 50개의 차이"보다 "이미지 1개와 5개의 차이"가 더 중요한 경우가 많은데,  
원래 수치로는 이 비율 관계를 반영하기 어렵습니다.

**해결 1: 로그 변환**  
텍스트 길이, 이미지 수, 가격 등 개수 성격의 수치 18가지에 로그 변환을 적용합니다.  
예) 이미지 100개 → log(101) ≈ 4.6, 이미지 10개 → log(11) ≈ 2.4  
이렇게 하면 극단적으로 큰 수치의 영향을 줄이고, 비율 관계를 더 잘 반영할 수 있습니다.

로그 변환 적용 대상 (18가지):

> 전체 텍스트 길이, 이미지 수, 표 수, 목록 항목 수, 단락 수, 섹션 수,  
> 구조화 데이터 항목 수, 구체적 숫자 표현 수, 모호한 표현 수,  
> 가격(원), 피부 타입 수, 주요 성분 수, 효능 키워드 수, 텍스처 키워드 수,  
> 부정적 표현 수, 인증 마크 수, 소셜 증거 요소 수, 외부 기관 인용 수

로그 변환 미적용 (비율·점수이므로 이미 적절한 범위):

> 수치 구체성 비율, AI 친화도 종합 점수

---

**문제 2: AI의 제시 순서 편향**  
AI에게 상품을 보여줄 때 첫 번째로 제시된 상품이 더 자주 1위를 받는 경향이 있습니다.  
이 정보 없이 학습하면 모델이 순서 편향을 보정하지 못합니다.

**해결 2: 제시 순서를 특징으로 추가**  
각 상품이 AI에게 몇 번째로 제시됐는지(1·2·3번)를 특징에 포함합니다.  
0, 0.5, 1로 정규화해 다른 특징과 크기를 맞춥니다.  
이 옵션은 켜고 끄기 가능합니다.

## 개선 방향 3: 모델 종류

모든 모델이 동일한 전처리(로그 변환, 정규화)와 동일한 평가 지표를 사용합니다.

### 다층 신경망 (기본 모델)

상품 특징을 입력받아 점수를 출력하는 3층 신경망입니다.

```
입력 특징 (20~21개)
  → 128개 노드 층
  → 64개 노드 층
  → 32개 노드 층
  → 점수 1개 출력
```

- 학습 파일: `src/train.py`
- 학습 과정 중 성능이 15번 연속 나아지지 않으면 자동으로 학습을 멈춥니다.

---

### LightGBM (트리 기반 모델)

결정 트리를 여러 개 쌓아 올리는 방식의 모델입니다.  
표 형식 데이터에서 신경망보다 성능이 좋은 경우가 많으며, 특히 학습 데이터가 적을 때 강점이 있습니다.

- 학습 파일: `src/lgbm_train.py`
- 최대 500개 트리를 학습하며, 성능이 20번 연속 나아지지 않으면 자동으로 멈춥니다.

---

### XGBoost (트리 기반 모델)

LightGBM과 유사한 트리 기반 모델로, 구현 방식이 다릅니다.  
같은 조건에서 두 모델을 모두 실험해 더 나은 쪽을 선택합니다.

- 학습 파일: `src/xgb_train.py`
- 최대 500개 트리를 학습하며, 성능이 20번 연속 나아지지 않으면 자동으로 멈춥니다.

## 모든 모델에 공통으로 적용되는 후처리: 온도 보정

모델이 출력하는 점수는 상대적인 숫자일 뿐, 확률이 아닙니다.  
이 점수를 "300개 상품 중 이 상품이 추천될 확률"로 변환하려면 보정 과정이 필요합니다.

**온도**라는 값으로 점수를 나눈 뒤 확률로 변환합니다.

- 온도가 낮으면: 1위 상품에 확률이 집중됩니다.
- 온도가 높으면: 전체 상품에 확률이 고르게 분산됩니다.

최적의 온도값(0.1, 0.2, 0.5, 0.7, 1.0, 1.5, 2.0, 3.0, 5.0 중)은 검증 데이터에서 가장 정확한 확률을 내는 값으로 자동 선택합니다.  
세 모델(신경망, LightGBM, XGBoost) 모두 동일한 방식으로 보정합니다.

## 평가 지표

| 지표 | 의미 | 참고 |
|------|------|------|
| 1위 정확도 | AI가 1위로 고른 상품을 모델도 1위로 예측한 비율 | 높을수록 좋음 |
| 쌍별 정확도 | 두 상품 비교 시 선호 방향을 맞힌 비율 | 높을수록 좋음 |
| 켄달 타우 | 전체 순위 순서 일치도 (-1 ~ 1) | 1에 가까울수록 좋음 |
| 음의 로그 우도 | 실제 선택에 얼마나 높은 확률을 부여했는지 | 낮을수록 좋음 |
| 브라이어 점수 | 예측 확률과 실제 결과의 차이 | 낮을수록 좋음 |

> **참고:** 3개 중 순위를 맞히는 문제에서 NDCG@3 지표는 무작위 예측만 해도 0.89 수준이 나와 변별력이 낮습니다. 위 5가지 지표로 모델 성능을 비교합니다.
