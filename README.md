# 폼클렌저 상품 페이지, AI가 추천하는 기준을 예측한다

> AI 챗봇이 폼클렌저 상품 페이지를 보고 추천 순위를 매길 때 **(1) 사용자의 질문 방식·타겟
> 소비자층에 따라 선호가 어떻게 달라지는지**, 그리고 **(2) 질문을 하나의 조건으로 고정했을 때
> 어떤 페이지 요소가 추천을 좌우하는지**를 학습하고, 새로운 상품이 AI 추천을 받을 가능성을
> 수치로 예측하는 머신러닝 프로젝트입니다.

---

## 1. 프로젝트 주제 및 문제 정의

### 배경

소비자들은 점점 AI 챗봇에 "피부 트러블이 있는데 어떤 폼클렌저가 좋아?" 같은 질문을 던집니다. AI 챗봇은 상품 페이지의 정보를 직접 읽고 추천 여부를 결정합니다. 이 말은 곧, **사용자가 어떤 의도로 질문하느냐, 또 상품 페이지를 어떻게 구성하느냐에 따라 AI 노출 가능성이 달라질 수 있다**는 뜻입니다.

### 핵심 질문

프로젝트는 두 개의 질문을 **이 순서로** 풉니다.

> **Q1. (질문 측면)** 같은 상품이라도 질문 유형(예: 가격 중심 vs 증상 중심)과 타겟 소비자층(페르소나)이 바뀌면 AI의 추천이 달라지는가? 달라진다면 그 효과를 학습해 예측할 수 있는가?
>
> **Q2. (상품 측면)** 그렇다면 질문을 하나의 조건으로 고정했을 때, AI는 페이지의 어떤 요소를 보고 추천 순위를 매기는가?

먼저 "질문이 추천을 바꾸는가"를 입증하고(Q1), 그 위에서 "조건을 고정하면 무엇이 추천을 좌우하는가"로 좁혀 들어갑니다(Q2). 질문 효과가 크다면, 피처 분석은 반드시 **특정 질문 조건을 고정한 뒤** 수행해야 의미가 있기 때문입니다.

> 참고: 분석 서사는 질문 측면(Q1)을 먼저 제시하지만, 실제 데이터 수집은 **질문 고정 조건이 먼저**였습니다(중간발표). 질문 변형 실험은 최종발표에서 추가했습니다(6장).

### 왜 데이터마이닝 문제인가

이 문제는 단순 분류가 아니라 **선호 학습(preference learning) / learning-to-rank** 문제입니다.

- 정답이 "추천/비추천"의 이진 라벨이 아니라 **상대적 순위**(3개 중 1·2·3위)로 주어집니다.
- 한 번의 관측이 "A>B, A>C, B>C" 세 개의 쌍별 비교를 동시에 담아, 수천 번 반복하면 수백 개 상품 사이의 잠재 선호 구조를 대규모로 복원할 수 있습니다.
- 학습한 모델에서 **어떤 변수가 선호를 가르는지**를 역으로 캐내는 것(feature importance mining)이 곧 실무 인사이트가 됩니다.
- 학습에 없던 새 브랜드에도 통하는지(일반화)가 성패를 가르는, 전형적인 데이터마이닝 검증 구조를 갖습니다.

## 2. 데이터 확보, 전처리 및 최종 데이터셋

### 2.1 데이터 수집: AI에게 직접 시험을 보게 했다

AI가 실제로 어떤 순서로 상품을 선호하는지 관찰하기 위해, AI에게 직접 순위를 매기게 했습니다.

```
폼클렌저 상품 3개를 묶어 AI에게 동시에 보여준다
         ↓
AI가 세 상품을 1위부터 3위까지 순위를 매긴다
         ↓
이 과정을 대규모로 반복한다
```

한 번의 실험은 단순해 보이지만 "A가 B보다, A가 C보다, B가 C보다 낫다"는 세 비교 정보를 동시에 담습니다.

### 2.2 두 개의 데이터셋

질문 효과를 보는 **질문 변형 데이터**가 핵심 실험이고, 질문을 고정한 **질문 고정 데이터**는 피처 중요도 분석의 "고정 조건" 역할을 합니다.

| 구분 | 질문 변형 (Q1, 핵심) | 질문 고정 (Q2, 피처 분석 조건) |
|------|----------------------|--------------------------------|
| 원본 파일 | `data/raw/anthropic_various_query_type.csv` | `data/raw/response.csv` |
| 질문 조건 | **변형** (질문 유형 5종 × 페르소나 4종) | **고정** (단일 프롬프트) |
| AI 엔진 | Anthropic (claude-haiku-4-5) | OpenAI, Anthropic |
| 상품 수 | 45개 | 295개 |
| 순위 실험 수 | 3,000건 | 약 4,200건 |
| 추가 입력 변수 | **query_type, persona** | 없음 |
| 수집 시점 | 최종발표(신규) | 중간발표 |

**질문 변형 데이터의 구조**: 각 실험은 `페르소나_질문유형_번호` 형태의 ID(예: `PRIMARY_PRC_027`)를 가지며,
- **질문 유형(query_type, 5종)**: USE(용도), CAT(카테고리), SYM(증상), DEC(성분/판단), PRC(가격)
- **페르소나(persona, 4종)**: PRIMARY, SECONDARY, TERTIARY1, TERTIARY2

같은 상품군을 질문 조건만 바꿔가며 반복 평가했습니다. 두 변수는 **한 실험 안의 세 상품에 공통**으로 적용됩니다.

### 2.3 입력 특징 (페이지 수치화 20종)

모델은 페이지 원문 대신, 페이지를 수치화한 20개 항목을 입력으로 받습니다.

| 분류 | 특징 항목 |
|------|----------|
| 텍스트 풍부도 | 전체 설명 길이, 단락 수, 섹션 수, 이미지 수, 표 수, 리스트 항목 수 |
| 신뢰도 표현 | 구체적 수치 표현 횟수, 모호한 표현 횟수, 수치의 구체성 비율 |
| 구조화 데이터 | 구조화된 정보 필드 수 |
| 성분 및 효능 | 핵심 성분 수, 효능 키워드 수, 질감 키워드 수, 부정 성분 표시 수 |
| 가격 | 판매 가격 |
| 피부 적합 정보 | 적합 피부 타입 수 |
| 인증 및 신뢰 | 인증 마크 수, 소셜 증거 요소 수, 외부 기관 인용 수 |
| 종합 평가 | 콘텐츠 신뢰도 종합 점수 |

처음 24개였으나 결측이 20%를 넘는 항목(pH 수치 98% 누락, 평점 39% 누락 등) 4개를 제거해 20개로 정리했습니다. 가격·이미지 수처럼 값의 범위가 큰 항목은 로그 변환으로 극단값의 영향을 줄였고, 결측은 **학습 fold에서 계산한 중앙값**으로만 대치해 정보 누출을 막았습니다.

### 2.4 의미 기반 임베딩 특징 (중간발표 계획 → 실행)

수치 특징만으로는 담기 어려운 설명의 "의미"를 반영하기 위해 텍스트·이미지를 임베딩으로 변환해 추가했습니다.

- **텍스트 임베딩**: BGE-M3 (1024차원) → PCA 16차원
- **이미지 임베딩**: CLIP (512차원) → PCA 8차원

고차원을 그대로 쓰면 적은 상품 수에 비해 과적합되므로, **학습 fold에서만 PCA를 학습**(검증/테스트는 그 PCA를 그대로 적용)해 차원을 축소한 뒤 이어 붙입니다.

### 2.5 질문 변수 인코딩 (질문 변형 데이터)

`query_type`(5종)·`persona`(4종)는 순서가 없는 범주형이므로 **원핫 인코딩**해 9개의 0/1 열로 추가했습니다.

> **핵심 설계 판단**: 질문 변수는 한 실험의 세 상품에서 모두 같은 값이라, 순위를 가르는 데 **단독으로는 기여하지 못합니다**(셋 다 같은 값이면 상쇄). 오직 **상품 특징과의 상호작용**을 통해서만 효과를 냅니다. 트리 모델과 신경망은 이 상호작용을 학습할 수 있으므로(질문 유형으로 분기 → 그 아래에서 가격으로 분기), 원핫 후 모델이 상호작용을 잡게 하는 방식을 택했습니다. 4-2절 결과가 이 판단이 옳았음을 보여줍니다.

### 2.6 최종 입력 차원과 데이터 분할

- 질문 고정 데이터: 수치 20 + 임베딩 24 + 제시 순서 1 = **45차원**
- 질문 변형 데이터: 위 45차원 + 질문 원핫 9 = **54차원**

분할은 **브랜드 단위 홀드아웃(unseen-item)** 방식으로, 일부 브랜드를 학습에서 완전히 제외해 테스트에만 둡니다. 단순 암기가 아니라 처음 보는 상품에도 통하는 패턴을 학습했는지 검증하기 위해서입니다.

## 3. 사용한 방법론 및 모델링 과정

### 3.1 왜 순위 확률 모델(Plackett-Luce)인가

정답이 순위이므로, 순위가 나올 확률을 직접 모델링합니다. 각 상품에 내재적 매력도 점수가 있다고 가정하면,

```
[A > B > C] 순서가 나올 확률
  = (A 점수) / (A+B+C)   ← 셋 중 A가 1위로 뽑힐 확률
  × (B 점수) / (B+C)     ← 남은 둘 중 B가 뽑힐 확률
```

이렇게 하면 어떤 순서든 발생 확률을 계산할 수 있고, 학습 목표는 **AI가 실제로 매긴 순위에 가장 높은 확률이 부여되도록** 점수를 조정하는 것입니다. 이진 분류로 환원하면 잃어버리는 "3개 동시 비교" 정보를 그대로 활용한다는 점이 선택 이유입니다.

### 3.2 왜 이 모델들인가 (모델 선택 근거)

| 모델 | 선택 이유 |
|------|-----------|
| **Plackett-Luce 점수 회귀** | 가장 단순한 선형 기준선. 복잡한 모델의 성능을 비교할 출발점이 필요했습니다. |
| **LightGBM / XGBoost (LambdaRank)** | 입력이 정형 수치 특징이고 상품 수가 적은(수십~수백) 환경에서는 트리 부스팅이 신경망과 대등하거나 우월합니다. 또 **질문 변수와 상품 특징의 상호작용**을 분기 구조로 자연스럽게 잡습니다(2-5절). |
| **심층 신경망(MLP)** | 임베딩처럼 연속·고차원 신호의 비선형 결합을 학습하는 데 강점. 트리와 오차 성격이 달라 블렌딩 다양성을 제공합니다. |
| **PL 목적함수 변형(lgbm_pl, xgb_pl)** | 순위 손실 대신 PL 매력도를 직접 회귀. 비교·보정용으로 유지했습니다. |
| **신규 pairwise 4종 — RandomForest · RankSVM · LogisticRegression · EBM** | 모델 다양성과 **해석 가능성**(EBM 효과 곡선, 로지스틱 계수) 확보용으로 추가. 순위가 아니라 **항목 쌍의 특징 차이 ΔX → 어느 쪽이 상위인지**를 맞히는 쌍 분류 방식입니다(4-3절). |

### 3.3 작은 데이터에서 일반화를 끌어올리는 세 장치

상품 수가 적어 한 번의 분할만으로는 결과가 운에 휘둘립니다. 그래서:

1. **Brand-CV 하이퍼파라미터 선택**: 작은 검증셋 하나에 맞추지 않고, 브랜드 기준 5겹 교차검증의 **5개 검증 점수 평균**으로 설정을 고릅니다. 선택 기준은 1위 정확도·순위 일치도·확률 보정 오차를 표준화해 합산한 **균형 점수**입니다.
2. **Seed 앙상블**: 같은 설정으로 seed만 바꿔 5개 모델을 학습해 점수를 평균, 무작위성에서 오는 분산을 줄입니다.
3. **모델 블렌딩**: MLP·LightGBM·XGBoost 점수를 표준화해 정렬한 뒤, 검증셋에서 최적 가중 평균을 탐색해 합칩니다.

### 3.4 위치 편향 보정과 확률 보정

- **위치 편향**: 먼저 제시된 상품이 더 자주 1위를 받는 경향이 있어, 각 상품의 제시 순서를 입력에 포함해 보정합니다.
- **온도 보정(calibration)**: 모델 점수는 그 자체로 확률이 아닙니다. 온도값으로 나눈 뒤 확률로 변환하며, 최적 온도는 검증셋에서 예측 확률이 가장 정확해지는 값을 탐색해 정합니다.

### 3.5 전체 처리 흐름

```
원본 상품 페이지 + 질문 조건(질문 변형 데이터)
       ↓
20개 수치 특징 추출 + 임베딩 PCA + 질문 원핫
       ↓
3개씩 묶어 AI에게 순위 요청
       ↓  ← 학습에만 사용
Plackett-Luce로 각 상품의 내재적 매력도 추정
       ↓
특징 → 랭킹 모델(트리/신경망) → 추천 점수
       ↓
브랜드 홀드아웃 테스트에서 성능 측정 + 온도 보정
```

## 4. 최종 모델링 결과 및 성능 분석

### 4.1 평가 지표 선택과 그 이유

| 지표 | 의미 | 왜 중요한가 | 무작위 수준 |
|------|------|-------------|:----------:|
| **1위 예측 정확도** | AI가 1등으로 고른 상품을 모델도 1등으로 맞힌 비율 | "AI가 무엇을 가장 먼저 추천하는가"가 노출의 핵심이므로 **주력 지표** | 33.3% |
| **쌍별 선호 정확도** | 두 상품 중 어느 쪽이 더 선호되는지 맞힌 비율 | 1위 외 전반적 선호 방향의 정확도 | 50.0% |
| NDCG@3 | 상위 항목을 얼마나 제대로 위에 놓았는지 | 3개 목록에서는 무작위로도 0.89가 나와 **변별력이 낮음**(보조 지표) | ~0.89 |
| 순위 일치도(Kendall τ) | 전체 순위가 실제와 얼마나 일치하는지 | 순서 전반의 품질 | 0.00 |
| 확률 보정 오차(NLL) / 신뢰도 오차(Brier) | 예측 확률이 현실과 얼마나 맞는지 | "추천 확률"을 의사결정에 쓰려면 **확률이 신뢰 가능해야** 함 | 1.79 / 0.67 |

정확도만으로는 부족하다고 보고, **순위 정확도(1위·쌍별) + 확률 신뢰도(NLL·Brier)** 를 함께 봅니다. NDCG@3은 포화되어 주력에서 제외했습니다.

### 4.2 Q1 — 질문 프레이밍은 추천을 바꾸는가 (질문 변형 데이터)

먼저 질문 효과 자체를 봅니다. 같은 "아이템 특징만" 모델이 질문 환경에 따라 어떻게 달라지는지를 같은 척도(1위 정확도)로 정렬하면:

| 조건 | 입력 | 1위 정확도 |
|------|------|:--------:|
| 무작위 | — | 0.333 |
| 질문 변형, 아이템 암기(인기순) 베이스라인 | 상품 ID만 | 0.590 |
| 질문 변형, **질문 숨김** | 아이템 특징만 | ~0.56 |
| 질문 변형, **질문 공개** | **+ query_type/persona** | **0.69** |

질문을 숨기면 "인기 제품 찍기(0.59)"조차 못 넘지만(0.56), 질문 문맥을 넣으면 0.69로 **+10%p** 회복됩니다. 다른 변수는 고정이므로 이 상승분은 **전적으로 질문 프레이밍 효과**입니다. 즉 같은 상품도 질문에 따라 순위가 바뀌며, 그 변화는 학습 가능합니다.

질문 변형 데이터 전체 모델 성능(새 브랜드 736건, Anthropic haiku). 디폴트 파라미터(baseline)와 brand-CV 튜닝 후:

| 모델 | baseline 1위 | 튜닝 후 1위 | Δ |
|------|:----------:|:---------:|:---:|
| **LightGBM** | 0.670 | **0.692** | +0.022 |
| XGBoost | 0.643 | 0.675 | +0.033 |
| MLP | 0.637 | 0.617 | -0.020 |
| lgbm_pl | 0.553 | 0.609 | +0.056 |
| xgb_pl | 0.556 | 0.558 | +0.003 |

튜닝된 LightGBM이 1위 정확도 0.692로 최고(쌍별 0.769, NDCG@3 0.959, Brier 0.414). 튜닝으로 트리 모델은 +2~3%p 개선됐으나, MLP는 튜너가 더 큰 망을 골라 이 작은 제품군에서 오히려 하락했습니다(과적합).

### 4.3 Q2 — 질문을 고정하면 무엇이 추천을 좌우하는가 (질문 고정 데이터)

### 4.4 하이퍼파라미터 튜닝 과정

각 모델의 설정은 **브랜드 기준 5겹 교차검증(brand-CV)** 으로 선택했습니다(`src/tune/*.py`, 공통 로직 `src/tune/cv_common.py`). 핵심은 작은 검증셋 하나에 과적합되지 않도록, 같은 후보를 5개 fold에서 평가해 **검증 점수의 평균**으로 고르는 것입니다.

```
하이퍼파라미터 후보 그리드
        ↓
각 후보를 브랜드 5겹 CV로 평가 (fold마다 다른 브랜드를 검증에 둠)
        ↓
fold 5개의 균형 점수 평균 = 1위 정확도 + 순위 일치도 + (-NLL) 표준화 합
        ↓
최고 균형 점수 후보 채택 → artifacts/tuning*/{model}_best_params.json
        ↓
선택된 설정으로 단일 분할 + 5-seed 앙상블 재학습 → 최종 test 평가
```

튜닝은 **시맨틱 → MLP → LightGBM → XGBoost → PL 변형 → 신규 pairwise 4종** 순으로 실행했고, 후보 수는 모델별로 LightGBM 36 · XGBoost 72 · MLP 36 · PL 변형 24~48 · 신규 4종 4~16개였습니다.

- **시맨틱 스윕**: `use_semantic_features`, 텍스트·이미지 PCA 차원을 LightGBM 프록시로 탐색. OpenAI 고정 데이터는 텍스트 16 + 이미지 8을, **Anthropic·질문 변형 데이터는 이미지 임베딩이 도움이 안 돼 텍스트만**(Anthropic 8 / 질문 변형 16) 채택했습니다.
- **선택된 트리 설정(대표)**: 얕은 트리(`max_depth=4~5`, `num_leaves=31`)와 강한 정규화(`min_child` 큰 값, `reg_lambda≥0.5`, `feature_fraction=0.7`) — 상품 수가 적어 과적합을 누르는 방향으로 수렴했습니다.
- **선택 기준의 일관성**: 모든 튜너가 동일한 균형 점수를 써서, "1위 정확도만 높고 확률 보정이 나쁜" 후보가 뽑히지 않도록 했습니다.

> **참고 — 4.2 baseline과 4.3 값이 다른 이유**: 두 절은 **서로 다른 데이터셋**입니다(질문 변형 45개 vs 질문 고정 295개). 또 4.2의 baseline 열은 **튜닝 전 디폴트**, 4.3 표는 **튜닝 후 최종** 값이라 절대 수치를 직접 비교할 수 없습니다.

## 5. 결과 해석

### 5.1 질문 프레이밍은 추천을 가른다 (질문 변형 데이터)

- 질문 문맥이 **아이템 암기 대비 +10%p**의 예측력을 더합니다(4-2절). 같은 상품도 질문에 따라 순위가 바뀝니다.
- 신호 크기는 **persona > query_type** — 타겟 소비자층이 질문 카테고리보다 추천을 더 강하게 가릅니다.
- 단, 이 모델이 예측하는 것은 인간 선호가 아니라 **LLM(추천 엔진)의 행동**입니다.

### 5.2 고정 조건에서 AI는 페이지의 무엇을 보는가 (사후 분석)

질문을 하나의 조건으로 고정한 뒤(질문 고정 데이터의 최고 모델 LightGBM), SHAP 분석으로 학습한 패턴을 역추적했습니다(`artifacts/analysis/ANALYSIS_RESULTS.md`).

- **가점 요인**: 인증 마크(있다/없다가 거의 이진 신호), 핵심 성분 3개 이상, EAT(콘텐츠 신뢰도) 종합 점수, 소셜 증거.
- **감점 요인**: 장황한 주장(claim) 위주 마케팅 텍스트, 가격의 **양극단**(중간 가격대 선호 + 초저가·초고가 감점의 비단조 효과).
- **콘텐츠 임베딩이 점수 기여의 60%**로 주된 동력이며, 임베딩 도입이 1위 정확도를 63%→70%로 끌어올렸습니다.
- **위치 편향은 거의 없음**(제시 순서 기여 0.9%) — AI 라벨이 제시 순서에 흔들리지 않아 라벨 품질 측면에서 긍정적.
- LightGBM·XGBoost의 중요도 순위 상관이 0.93으로, 단일 모델의 우연이 아닙니다. 신규 EBM·로지스틱 모델의 효과 방향도 같은 신호를 가리켜 해석을 보강합니다.

> **분석 조건에 대한 단서**: 이 피처 중요도는 **질문 고정 데이터(OpenAI, 상품 295개)** 위에서 산출한 것으로, "하나의 고정된 질문 조건"에 해당합니다. 질문 변형 데이터(haiku, 45개)의 특정 (질문×페르소나) 슬라이스에서 다시 돌린 것은 아니며, 둘은 엔진·상품 풀이 다릅니다. 따라서 위 결론은 "고정 조건에서 AI가 보는 페이지 요소"의 대표적 사례로 읽어야 합니다.

### 5.3 실제 문제에 대한 의미: 세그먼트 조건부 전략

종합하면, AI 추천은 **타겟 소비자(페르소나)와 질문 프레이밍에 따라 체계적으로 달라집니다.** 따라서 "범용 추천 예측기" 하나보다, **회사가 타겟하는 세그먼트별로 조건부 모델을 만드는 것**이 올바른 접근입니다.

- 질문 변수가 입력이므로 "PRIMARY 페르소나 × 가격 질문" 같은 특정 슬라이스 예측을 바로 뽑을 수 있습니다.
- 비즈니스 산출물은 1위 정확도 숫자가 아니라, **그 세그먼트에서 AI 추천 순위를 올리는 페이지 개선 레버**(인증·성분·콘텐츠 신뢰도 강화, 과장 텍스트 축소 등)입니다. 이는 검색 최적화(SEO)의 AI 버전, 즉 GEO(생성형 엔진 최적화)에 해당합니다.

## 7. 한계점 및 향후 개선 방향

### 한계

- **제품 풀 규모**: 질문 변형 데이터는 상품 45개로, 임의의 신제품으로의 일반화는 보장하기 어렵습니다(질문 효과 학습에는 충분하나 새 상품 점수화는 별개).
- **질문 격자의 제약**: 학습한 5개 질문 유형 × 4개 페르소나 **밖의 프레이밍은 외삽 불가**. 새 프레이밍은 데이터를 새로 수집해야 합니다.
- **쌍 비교 모델의 일반화 격차**: 신규 pairwise 4종은 CV 대비 미지 브랜드 test에서 크게 하락해(4-3절), 소규모 순위 문제에서는 직접 순위 목적함수(트리)가 더 견고함을 확인했습니다.
- **성능 천장 미측정**: 0.69가 LLM의 자기 일관성(같은 프롬프트 반복 시 같은 답을 내는 정도) 한계 대비 어디인지 알 수 없습니다(반복 샘플 데이터 부재).
- **단일 모델·단일 엔진**: 질문 변형 실험은 claude-haiku 한 모델. 더 강한 모델일수록 더 결정론적이라 예측 가능성이 달라질 수 있습니다.
- **연관 ≠ 인과**: SHAP 중요도는 모델이 의존하는 신호이지 AI의 실제 추론 근거와 동일하지 않습니다.

### 향후 방향

- **세그먼트 조건부 데이터 수집**: 타겟 페르소나·질문으로 LLM 랭킹 데이터를 모아 같은 파이프라인으로 조건부 모델 학습(5-3절 레시피).
- **제품 풀 확대 + 강한 엔진**으로 새 상품 일반화 검증.
- **자기 일관성 천장 측정**: 동일 프롬프트 반복 샘플링으로 달성 가능한 상한을 정량화.
- **질문 × 특징 명시적 상호작용항** 추가, 페르소나별 온도 분리 등 변별력 강화.
- 죽은 특징(예: 외부 권위 인용 수) 정리 및 PL 목적함수의 수치 안정화.

## 부록 B. 실행 방법

패키지 설치 후, 모든 명령은 **저장소 루트**에서 실행합니다.

```bash
pip install -r requirements.txt
```

**질문 고정 데이터 (Q2) — 튜닝 → 학습 → 블렌딩**

```bash
python src/tune/tune_lgbm.py        # brand-CV 하이퍼파라미터 → artifacts/tuning*/
python src/lightgbm/lgbm_train.py   # seed 앙상블 학습 → artifacts/preds*/
python src/blend.py                 # 세 모델 블렌딩
```

**질문 변형 데이터 (Q1) — `DM_DATASET=query_type` 한 줄로 전환**

```bash
python src/prep_query_type.py                       # 데이터셋 생성
DM_DATASET=query_type python src/pl_fitting.py      # PL 라벨
DM_DATASET=query_type python src/tune/tune_lgbm.py  # 튜닝 → artifacts/tuning_step2qt/
DM_DATASET=query_type python src/lightgbm/lgbm_train.py   # 학습 → artifacts/preds_step2qt/
```

**신규 pairwise 모델 4종**

```bash
# handoff_new_models/ 에서 튜닝 → 학습. Anthropic 변형은 DM_ENGINE=anthropic
python handoff_new_models/tune/tune_rf.py
python handoff_new_models/rforest/rf_train.py
```

환경변수 `DM_DATASET=query_type`를 주면 모든 학습/튜닝 스크립트가 질문 변형 데이터(`step2qt` 프로토콜, 54차원 입력, 별도 산출물 폴더)로 전환됩니다. 튜닝 JSON이 없어도 각 스크립트는 정규화된 기본 설정으로 동작합니다. 학습 로그는 Weights & Biases로 확인하며, 오프라인은 `WANDB_MODE=offline`. 버전별 기록은 `EXPERIMENTS.md`를 참고하세요.
