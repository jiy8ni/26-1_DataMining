# 폼클렌저 상품 페이지, AI가 추천하는 기준을 예측한다

> AI 챗봇이 폼클렌저 상품 페이지를 보고 추천 여부를 결정할 때 어떤 기준을 따르는지 학습하고,
> 새로운 상품 페이지가 AI에게 선택될 가능성을 수치로 예측하는 머신러닝 프로젝트입니다.

---

## 왜 이 문제인가

소비자들은 점점 AI 챗봇에 "피부 트러블이 있는데 어떤 폼클렌저가 좋아?" 같은 질문을 던집니다. AI 챗봇은 상품 페이지의 정보를 직접 읽고 추천 여부를 결정합니다. 이 말은 곧, **상품 페이지를 어떻게 구성하느냐에 따라 AI 노출 가능성이 달라질 수 있다**는 뜻입니다.

이 프로젝트는 하나의 질문에서 출발합니다.

> "AI가 상품 페이지를 보고 추천 순위를 매길 때, 어떤 요소를 중시하는가?"

이 패턴을 파악할 수 있다면, 새로 출시되는 상품이 AI 추천을 받을 가능성을 미리 예측하거나 상품 페이지 개선 방향을 제시하는 것이 가능합니다.

---

## 데이터 수집: AI에게 직접 시험을 보게 했다

AI가 실제로 어떤 순서로 상품을 선호하는지 관찰하기 위해, 다음 방식으로 데이터를 수집했습니다.

```
폼클렌저 상품 3개를 무작위로 묶어 AI에게 동시에 보여준다
         ↓
AI가 세 상품을 1위부터 3위까지 순위를 매긴다
         ↓
이 과정을 약 4,200회 반복한다
```

한 번의 실험은 단순해 보이지만, "상품 A가 B보다 낫다", "A가 C보다 낫다", "B가 C보다 낫다"는 세 가지 비교 정보를 동시에 담고 있습니다. 이 방식으로 **295개 상품** 사이의 상대적 선호 관계를 대규모로 수집했습니다.

두 종류의 AI 엔진(OpenAI, Anthropic)을 사용해 각각 독립적으로 실험했으며, 서로 다른 엔진의 판단을 섞을 경우 학습 신호가 충돌해 모델 성능이 저하됨을 확인했습니다. 현재 실험은 OpenAI 데이터를 기준으로 진행합니다.

---

## 상품 페이지에서 추출한 입력 특징 (20가지)

모델은 상품 페이지 원문 대신, 페이지를 수치화한 20개 항목을 입력으로 받습니다.

| 분류 | 특징 항목 |
|------|----------|
| 텍스트 풍부도 | 전체 설명 길이, 단락 수, 섹션 수, 이미지 수, 표 수, 리스트 항목 수 |
| 신뢰도 표현 | 구체적 수치 표현 횟수, 모호한 표현 횟수, 수치의 구체성 비율 |
| 구조화 데이터 | 구조화된 정보 필드 수 |
| 성분 및 효능 | 핵심 성분 수, 효능 관련 키워드 수, 질감 관련 키워드 수, 부정 성분 표시 수 |
| 가격 | 판매 가격 |
| 피부 적합 정보 | 적합 피부 타입 수 |
| 인증 및 신뢰 | 인증 마크 수, 소셜 증거 요소 수, 외부 기관 인용 수 |
| 종합 평가 | 콘텐츠 신뢰도 종합 점수 |

처음 수집했을 때 24개 항목이었으나, 전체 상품의 20% 이상에서 데이터가 누락된 항목(pH 수치 98% 누락, 평점 39% 누락 등) 4가지를 제거해 20개로 정리했습니다.

가격, 이미지 수처럼 값의 범위가 큰 항목은 로그 변환을 적용해 극단적으로 큰 값이 모델에 과도한 영향을 주지 않도록 처리했습니다.

### 의미 기반 임베딩 특징 추가

수치 특징만으로는 담기 어려운 상품 설명의 "의미"를 반영하기 위해, 상품 페이지 텍스트와 대표 이미지를 임베딩 벡터로 변환해 입력에 추가했습니다.

- **텍스트 임베딩**: BGE-M3 (1024차원)
- **이미지 임베딩**: CLIP (512차원)

고차원 임베딩을 그대로 쓰면 학습 상품 수(약 259개)에 비해 차원이 너무 커 과적합됩니다. 그래서 **학습 fold에서만 PCA를 학습**(검증/테스트에는 그 PCA를 그대로 적용해 정보 누출 방지)해 텍스트는 16차원, 이미지는 8차원으로 축소한 뒤 기존 특징에 이어 붙입니다.

최종 입력은 **20개 수치 특징 + 24개 임베딩 차원 + 제시 순서 1개 = 45차원**입니다. 임베딩 사용 여부는 설정(`use_semantic_features`)으로 끄고 켤 수 있어, 수치 특징만 쓰는 기준선과 비교할 수 있습니다.

## 학습 핵심 원리: 순위를 확률로 표현하기

### Plackett-Luce 모델

각 상품에 내재적인 매력도 점수가 있다고 가정합니다. 세 상품이 경쟁할 때 특정 순서가 나올 확률은 다음처럼 계산됩니다.

```
[상품 A > 상품 B > 상품 C] 순서가 나올 확률

  = (A의 점수) / (A + B + C)    ← A가 세 개 중 1위로 선택될 확률
  × (B의 점수) / (B + C)        ← 남은 두 개 중 B가 선택될 확률
```

이 공식을 사용하면 어떤 순위 순서든 그 발생 확률을 계산할 수 있습니다. 모델 학습의 목표는 AI가 실제로 매긴 순위에 가장 높은 확률이 부여되도록 각 상품의 점수를 조정하는 것입니다.

### 위치 편향 보정

실험 결과, 세 상품 중 첫 번째로 제시된 상품이 더 자주 1위를 받는 경향이 발견됐습니다. 이 편향을 보정하기 위해 각 상품이 몇 번째로 제시됐는지를 입력 특징에 포함했습니다.

## 전체 처리 흐름

```
원본 상품 페이지
       ↓
20가지 수치 특징 추출
       ↓
3개씩 묶어 AI에게 순위 요청 (약 3,400건)
       ↓  ← 학습에만 사용
Plackett-Luce 모델로 각 상품의 내재적 매력도 추정
       ↓
특징(20개) → 학습 모델 → 추천 점수 출력
       ↓
전체 295개 상품에 점수, 순위, 추천 확률 부여
```

## 일반화 평가: 처음 보는 브랜드에도 통하는가

학습 데이터를 잘 맞히는 것보다, **학습 중 한 번도 등장하지 않은 브랜드**의 상품도 잘 예측하는지가 핵심 평가 기준입니다. 단순 암기가 아니라 진짜 패턴을 학습했는지 검증하기 위해서입니다.

전체 브랜드 중 약 10개를 처음부터 완전히 분리해두었습니다.

```
전체 데이터
  ├── 학습용: 대부분의 브랜드, 3,384건의 순위 실험
  ├── 검증용: 일부 브랜드, 279건
  └── 테스트용: 분리된 새 브랜드, 818건  ← 학습 중 절대 보지 않음
```

최종 성능은 이 테스트 데이터에서만 측정합니다.

## 실험한 모델

### 기준 모델: Plackett-Luce 점수 회귀

순위 데이터로부터 각 상품의 매력도 점수를 직접 추정한 뒤, 그 점수를 예측 목표로 삼아 회귀 모델을 학습합니다. 학습한 매력도와 상품 페이지 특징 사이의 관계를 선형 방정식으로 근사합니다. 구조가 단순해 복잡한 모델의 성능을 비교할 기준점이 됩니다.

### 심층 신경망

상품 특징을 입력받아 점수를 출력하는 신경망입니다.

```
입력 (45차원: 수치 20 + 임베딩 24 + 제시 순서 1)
    → 64개 처리 노드
    → 32개 처리 노드
    → 추천 점수 1개 출력
```

각 배치에서 시험 하나(3개 상품)의 예측 순위와 실제 AI 순위 사이의 Plackett-Luce 손실을 최소화하며 학습합니다. 검증 데이터에서 성능 개선이 없으면 자동으로 학습을 멈추고, 가장 좋았던 시점의 모델을 저장합니다.

학습 상품이 약 259개로 적어 큰 신경망은 쉽게 과적합됩니다. 그래서 은닉층을 줄이고(64→32), 드롭아웃과 가중치 감쇠를 강화한 보수적인 구조를 기본값으로 둡니다. 구체적인 구조·정규화 값은 아래의 brand-CV 하이퍼파라미터 선택으로 결정합니다.

### 트리 기반 앙상블: 5겹 교차검증 (LightGBM, XGBoost)

수많은 결정 트리를 순서대로 쌓아올리는 방식으로, 정형 데이터에서 신경망과 자주 경쟁하는 모델입니다.

브랜드 기준으로 전체 데이터를 5등분해 모델 5개를 독립적으로 학습한 뒤, 최종 예측은 5개의 점수를 평균합니다. 이렇게 하면 학습에 사용할 수 있는 데이터가 늘어나고, 특정 브랜드 조합에 과적합되는 위험도 줄어듭니다.

```
전체 데이터 (브랜드 기준 5등분)
  ├── 모델 1: 1번 그룹 검증, 나머지로 학습
  ├── 모델 2: 2번 그룹 검증, 나머지로 학습
  ├── ...
  └── 모델 5: 5번 그룹 검증, 나머지로 학습
         ↓
  5개 모델 점수 평균 → 최종 추천 점수
```

## 일반화를 높이는 세 가지 기법

학습에 쓸 수 있는 상품이 약 259개로 적고, 검증 데이터도 279건으로 작아 한 번의 분할만으로는 결과가 운에 휘둘립니다. 이를 보완하기 위해 단일 분할(vanilla) 모델에 다음 세 가지를 적용합니다.

### 1. Brand-CV 하이퍼파라미터 선택

작은 검증 데이터 하나에 맞춰 설정을 고르면 그 데이터의 노이즈에 과적합됩니다. 대신 브랜드 기준 5겹 교차검증으로 같은 하이퍼파라미터 후보를 평가하고, **5개 검증 점수의 평균**으로 최적값을 고릅니다. 선택 기준은 1위 정확도·순위 일치도·확률 보정 오차를 표준화해 합산한 균형 점수입니다. 이렇게 고른 설정은 교차검증을 "선택 도구"로만 쓰고, **최종 산출물은 여전히 단일 분할 모델**입니다.

### 2. Seed 앙상블

같은 데이터·같은 설정으로도 초기값과 무작위성(트리 모델의 행/열 서브샘플링, 신경망의 초기 가중치)에 따라 모델이 조금씩 달라집니다. 서로 다른 seed로 5개 모델을 학습해 점수를 평균하면 이 분산이 줄어 결과가 안정됩니다.

### 3. 모델 블렌딩

신경망과 트리 모델은 서로 다른 종류의 오차를 냅니다. 세 모델(MLP·LightGBM·XGBoost)의 점수를 표준화해 정렬한 뒤, 검증 데이터에서 가장 좋은 가중 평균 비율을 탐색해 합칩니다. 보통 단일 최고 모델보다 안정적인 성능을 냅니다.

## 모델 출력: 점수를 확률로 보정하기

모델이 출력하는 점수는 그 자체로 "확률"이 아닙니다. 이 점수를 "295개 상품 중 이 상품이 선택될 확률"로 변환하려면 보정이 필요합니다.

온도라는 조정값으로 점수를 나눈 뒤 확률로 변환하는데, 온도가 낮으면 상위 상품에 확률이 집중되고 높으면 전체에 고르게 분산됩니다. 최적의 온도값은 검증 데이터에서 예측 확률이 가장 정확한 값을 자동으로 탐색해 결정합니다.

| 지표 | 의미 | 방향 | 무작위 예측 수준 |
|------|------|:----:|:---------------:|
| 1위 예측 정확도 | AI가 1등으로 고른 상품을 모델도 1등으로 맞힌 비율 | 높을수록 좋음 | 33.3% |
| 쌍별 선호 정확도 | 두 상품 중 어느 쪽이 더 선호되는지 맞힌 비율 | 높을수록 좋음 | 50.0% |

NDCG@3은 3개짜리 목록에서는 무작위 예측으로도 0.89 수준이 나와 모델 간 변별력이 낮습니다. 1위 정확도와 쌍별 정확도가 핵심 지표입니다.

| LightGBM | **72.9%** | **78.3%** | **0.96** | **0.57** | **1.41** | **0.40** |

모든 수치는 학습 중 한 번도 등장하지 않은 새 브랜드 상품 818건을 기준으로 측정했습니다. MLP·XGBoost·LightGBM은 brand-CV로 고른 하이퍼파라미터를 단일 분할 + 5-seed 앙상블로 다시 학습한 최종 test 결과입니다. 확률 보정 오차와 예측 신뢰도 오차는 순위 정확도와 별개로, 모델이 출력하는 확률값이 얼마나 현실에 맞게 보정되었는지를 나타내며, 이번 재실행에서는 LightGBM이 전 지표에서 가장 좋았습니다.

### 하이퍼파라미터 튜닝(brand-CV) 결과

아래 표는 `src/tune/tune_mlp.py`, `src/tune/tune_xgb.py`, `src/tune/tune_lgbm.py`가 저장한 최종 선택 후보의 **5겹 brand-CV 검증 평균 성능**입니다. 위의 test 성능표와 직접 비교하는 값이 아니라, 튜닝 단계에서 어떤 후보가 선택됐는지를 보여주는 참고 지표입니다.

| 튜닝 스크립트 | 균형 점수 | 1위 예측 정확도 | 쌍별 선호 정확도 | 추천 목록 품질 | 순위 일치도 | 확률 보정 오차 | 예측 신뢰도 오차 | 평균 온도 |
|------|:---------:|:--------------:|:---------------:|:-------------:|:----------:|:-------------:|:--------------:|:--------:|
| `tune_mlp.py` | 3.61 | 67.5% | 77.0% | 0.96 | 0.54 | **1.30** | 0.44 | 0.50 |
| `tune_xgb.py` | 3.95 | 66.8% | 75.8% | 0.95 | 0.52 | 1.35 | 0.45 | 0.50 |
| `tune_lgbm.py` | **4.28** | **69.6%** | **77.2%** | **0.96** | **0.54** | 1.31 | **0.43** | 0.32 |

세 튜닝 스크립트는 모두 `cv_common.py`의 동일한 균형 점수(1위 정확도·순위 일치도·NLL 조합)를 사용해 최종 후보를 고릅니다. 현재 설정에서는 LightGBM 튜너가 가장 높은 균형 점수를 기록했고, NLL만 놓고 보면 MLP 튜너가 가장 낮았습니다.

## 실행 방법

패키지 설치:

```bash
pip install -r requirements.txt
```

모든 명령은 **저장소 루트**에서 실행합니다 (데이터 경로가 루트 기준 상대 경로입니다).

기준 모델과 교차검증 앙상블:

```bash
python src/pl_regression.py          # Plackett-Luce 점수 회귀 (기준 모델)
python src/lightgbm/lgbm_kfold.py    # LightGBM (5겹 교차검증 앙상블)
python src/xgboost/xgb_kfold.py      # XGBoost (5겹 교차검증 앙상블)
```

단일 분할(vanilla) 모델 — 권장 순서 (튜닝 → 학습 → 블렌딩):

```bash
# 1) brand-CV로 하이퍼파라미터 선택 → artifacts/tuning/*.json
python src/tune/tune_lgbm.py
python src/tune/tune_xgb.py
python src/tune/tune_mlp.py

# 2) 선택된 설정으로 seed 앙상블 학습 → artifacts/preds/*.npz
python src/lightgbm/lgbm_train.py
python src/xgboost/xgb_train.py
python src/mlp/train.py

# 3) 세 모델 점수 블렌딩
python src/blend.py
```

튜닝 JSON이 없어도 각 학습 스크립트는 정규화된 기본 설정으로 동작합니다. seed 수는 `config.py`의 `n_seeds`로 조정하며, `n_seeds=1`로 두면 앙상블을 끄고 단일 모델 효과만 볼 수 있습니다.

기본 설정은 학습 중 한 번도 보지 않은 새 브랜드 상품에 대한 예측 방식으로 실행됩니다. 학습 진행 상황은 Weights & Biases를 통해 실시간으로 확인할 수 있으며, 처음 실행 시 터미널에서 로그인 안내가 표시됩니다 (오프라인으로 쓰려면 `WANDB_MODE=offline`). 버전별 실험 기록은 `EXPERIMENTS.md`를 참고하세요.
