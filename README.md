# 폼클렌저 상품 페이지 AI 추천 가능성 예측 모델

> AI 챗봇이 폼클렌저를 추천할 때 어떤 상품 페이지를 더 선호하는지 학습하고,  
> 새로운 상품 페이지가 AI에게 추천될 가능성이 얼마나 높은지 예측하는 모델입니다.

---

## 이 프로젝트가 해결하려는 문제

요즘 소비자들은 상품을 구매하기 전에 AI 챗봇에게 "피부 트러블이 있는데 어떤 폼클렌저가 좋아?"처럼 추천을 요청하는 경우가 늘고 있습니다.

AI 챗봇은 상품 페이지의 정보(성분 설명, 가격, 리뷰 수, 이미지 구성 등)를 보고 추천 여부를 결정합니다. 따라서 **상품 페이지를 어떻게 구성하느냐에 따라 AI에게 추천될 가능성이 달라질 수 있습니다.**

이 프로젝트는 다음 질문에 답하는 것을 목표로 합니다.

> "이 폼클렌저 상품 페이지는 AI에게 추천될 가능성이 얼마나 높은가?"

---

## 어떻게 학습 데이터를 만들었나

AI에게 폼클렌저 상품 3개를 동시에 보여주고, 어떤 순서로 추천하는지 순위를 매기게 합니다.

```
상품 A, 상품 B, 상품 C를 AI에게 제시
→ AI가 1위, 2위, 3위를 선택
예) A > C > B
```

이 과정을 수백 번 반복해 학습 데이터를 수집했습니다. 데이터에는 "A가 B보다 더 선호된다", "C가 B보다 더 선호된다"와 같은 **상대적 선호 정보**가 담겨 있습니다.

---

## 모델이 학습하는 것

각 상품 페이지의 특징(가격, 성분 수, 리뷰 수, 텍스트 길이 등 24가지 수치)을 입력받아, 그 상품이 AI에게 얼마나 선호될 가능성이 있는지를 나타내는 **점수(score)**를 출력합니다.

모델은 AI가 실제로 매긴 순위와 자신이 예측한 점수 순서가 최대한 일치하도록 학습합니다. 이를 위해 **Plackett-Luce 손실 함수**를 사용합니다.

Plackett-Luce 손실 함수는 "3개 중 어떤 것이 먼저 선택될지"를 확률로 표현합니다. 예를 들어 AI가 A > C > B 순서를 선택했다면, 모델이 이 순서에 높은 확률을 부여할수록 잘 학습된 것으로 봅니다.

---

## 최종 결과물

학습이 끝난 모델에 상품 페이지 정보를 입력하면 아래와 같은 결과를 얻을 수 있습니다.

| 지표 | 의미 | 예시 |
|------|------|------|
| 추천 점수 | 상대적 선호도를 나타내는 모델 점수 | 2.10 |
| 전체 순위 | 300개 상품 중 몇 위인지 | 12위 / 300개 |
| 상위 퍼센트 | 전체 중 상위 몇 퍼센트인지 | 상위 4% |
| 추천 확률 | 300개 중 이 상품이 선택될 확률 | 1.8% |
| 평균 대비 배율 | 평균적인 상품보다 몇 배 추천될 가능성이 높은지 | 3.1배 |

---

## 평가 방법

모델이 얼마나 잘 학습되었는지는 두 가지 관점으로 평가합니다.

**순위 정확도**: AI가 1위로 고른 상품을 모델도 1위로 예측했는가, 두 상품 간 선호 방향을 얼마나 맞혔는가

**일반화 성능**: 학습에서 한 번도 본 적 없는 새로운 브랜드 상품도 잘 예측하는가 (10개 브랜드를 별도로 보류해 테스트)

## 폴더 구조

```
├── src/                        모델 코드
│   ├── config.py               학습에 사용하는 모든 설정값 (학습률, 구조 등)
│   ├── data.py                 데이터를 불러오고 학습 형태로 변환
│   ├── model.py                추천 점수를 계산하는 신경망 모델
│   ├── loss.py                 Plackett-Luce 손실 함수
│   ├── calibration.py          점수를 확률로 변환하는 보정 과정
│   ├── metrics.py              모델 성능 평가 지표
│   ├── inference.py            300개 상품 전체에 순위와 확률 부여
│   └── train.py                학습 실행 진입점
│
├── data/
│   ├── processed/              모델 학습에 바로 사용할 수 있는 형태의 데이터
│   └── splits/                 학습 / 검증 / 테스트 분할 정보
│
├── notebooks/
│   └── 00_eda_and_split.ipynb  데이터 탐색 및 분할 과정
│
└── artifacts/eda/              탐색적 데이터 분석 결과물 (표, 시각화)
```

> `data/raw/` 폴더(원본 수집 데이터)는 용량 문제로 저장소에 포함되지 않습니다.

## 실행 방법

**필요한 패키지 설치**

```bash
pip install torch numpy pandas scikit-learn scipy wandb
```

**학습 실행**

```bash
cd src
python train.py
```

기본 설정은 학습에서 본 상품의 새로운 조합을 예측하는 방식(seen-item)으로 실행됩니다.  
학습에서 보지 않은 새 브랜드를 테스트하는 방식(unseen-item)으로 바꾸려면 `config.py`에서 아래 값을 수정합니다.

```python
protocol: str = "step2"   # "step1"(기본) → "step2"로 변경
```

학습 진행 상황은 [Weights & Biases](https://wandb.ai)에서 실시간으로 확인할 수 있습니다.  
처음 실행 시 터미널에서 로그인 안내가 표시됩니다.

## 데이터 수집 방식 요약

| 단계 | 내용 |
|------|------|
| 1단계 | 폼클렌저 300개 상품 페이지 수집 |
| 2단계 | 상품 페이지에서 24가지 수치 특징 추출 |
| 3단계 | 상품 3개씩 묶어 AI에게 순위 요청 (약 3,000회) |
| 4단계 | 수집한 순위 데이터로 모델 학습 |
| 5단계 | 전체 300개 상품에 추천 점수와 순위 부여 |
