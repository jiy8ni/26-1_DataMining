# EDA 안내서

이 문서는 앞으로 이 프로젝트의 탐색적 데이터 분석을 맡게 될 팀원을 위한 안내서입니다.
처음 보는 사람도 바로 이해할 수 있도록, 데이터 구조, 이미 확인된 이슈, 해석할 때 주의할 점, 다시 확인하는 방법을 한 번에 정리했습니다.

## 먼저 알아둘 핵심 한 줄

현재 데이터와 학습 파이프라인은 그대로 써도 돌아가지만, 결과를 해석할 때는 주석이 필요합니다.
특히 전처리 기준, 누락값이 많은 항목, 두 엔진의 응답 차이, 후보 제시 순서 효과, 상품 노출 불균형을 반드시 같이 봐야 합니다.

## 폴더 구조

- `data/raw`
  상품 페이지 원본 정보와 인공지능 응답 원본이 있습니다.
- `data/processed`
  학습용으로 정리된 파일이 있습니다.
- `artifacts/eda/step1`
  `step1` 기준 분석 결과입니다.
- `artifacts/eda/step2`
  `step2` 기준 분석 결과입니다.
- `artifacts/eda/response_diagnostics`
  인공지능 응답 자체를 진단한 결과입니다.

## 데이터 구조를 아주 간단히 설명하면

- `page_features.csv`
  각 상품 페이지에서 뽑은 숫자형 정보 모음입니다.
  예를 들면 가격, 용량, 텍스트 길이, 리뷰 수 같은 값이 들어 있습니다.
- `response.csv`
  인공지능이 세 개 후보 중 무엇을 1위, 2위, 3위로 골랐는지 기록한 파일입니다.
  같은 후보 세트에 대해 `OpenAI`와 `Anthropic`이 각각 한 번씩 응답했습니다.
- `trial_items.csv`
  각 후보 세트의 1번, 2번, 3번 자리에 어떤 상품이 들어갔는지 정리한 연결표입니다.

## `step1`과 `step2`를 어떻게 이해하면 되는가

- `step1`
  의도상으로는 이미 본 상품이 다른 조합으로 다시 나왔을 때 순위를 얼마나 잘 맞추는지 보는 설정입니다.
  다만 분할 기준이 상품 자체가 아니라 후보 세트 단위이기 때문에, 로직만 놓고 보면 어떤 상품이 우연히 학습용에 한 번도 안 들어가고 검증용이나 테스트용에만 남을 가능성을 완전히 배제하지는 못합니다.
  이번에 생성된 현재 `step1` 분할에서는 검증용과 테스트용에만 있고 학습용에는 없는 상품이나 브랜드가 확인되지는 않았습니다.
  그래도 이 결과를 곧바로 "새로운 상품에도 잘 일반화된다"라고 해석하면 과장입니다.
- `step2`
  학습 때 보지 않은 상품이나 브랜드가 들어왔을 때 얼마나 버티는지 보려는 설정입니다.
  일반화 성능을 보려면 이쪽을 더 중요하게 봐야 합니다.

## EDA를 할 때 기본 원칙

전체 데이터를 봐도 되는 분석과, 학습용 데이터만 봐야 하는 분석을 구분해야 합니다.

- 전체 데이터를 봐도 되는 것
  데이터 수집 품질 확인, 시장 대표성 확인, 후보 세트 구성 품질 확인
- 학습용 데이터만 봐야 하는 것
  순위 패턴 해석, 상품 선호 해석, 단순 기준선 만들기, 특징별 순위 차이 분석

쉽게 말하면, "데이터가 제대로 모였는가"는 전체로 봐도 되고, "모델이 무엇을 배울 것 같은가"는 학습용 데이터만 봐야 합니다.

## 이미 확인된 중요한 이슈

### 1. 검증용 데이터와 테스트용 데이터가 전처리에 조금 섞일 수 있음

현재 학습 데이터 준비 코드에서는 누락값을 채울 때, 학습용 데이터뿐 아니라 검증용 데이터와 테스트용 데이터 안에서 각각 가운데값을 다시 계산합니다.
이 방식은 평가 데이터의 분포를 전처리에 반영하게 만듭니다.

왜 중요한가:
평가 점수가 실제보다 조금 더 좋아 보일 수 있습니다.
또한 학습용, 검증용, 테스트용 데이터가 서로 다른 기준으로 채워질 수 있습니다.

어디서 확인하나:
`src/data.py`

### 2. 실제 학습에 들어간 표본 수가 원본 파일 행 수보다 적음

일부 행은 자동으로 빠집니다.

- 상품 매핑이 불확실한 경우 제외
- 세 개 후보가 모두 남지 않은 경우 제외
- 순위 정보가 불완전한 경우 사실상 사용하기 어려움

이미 확인된 수치:

- 원본 응답 파일에서 3위 정보가 비어 있는 행이 82개 있음
- `step1` 학습 파일에서 매핑이 불확실한 비율이 약 2.08%

왜 중요한가:
겉으로 보이는 파일 크기와 실제 학습 데이터 크기가 다릅니다.
그래서 "몇 개를 학습했다"라고 말할 때는 원본 행 수가 아니라 실제 사용된 후보 세트 수를 기준으로 말해야 합니다.

어디서 확인하나:

- `src/data.py`
- `artifacts/eda/step1/summary.json`
- `artifacts/eda/step2/summary.json`

### 3. 누락값이 너무 많은 항목이 그대로 모델 입력에 들어감

현재 모델 입력에는 누락이 매우 많은 항목도 포함되어 있습니다.

이미 확인된 수치:

- 산도 관련 값은 약 98% 누락
- 평점과 리뷰 수는 약 37%에서 40% 누락
- 용량은 약 18%에서 19% 누락

왜 중요한가:
이런 값은 실제 신호라기보다 "누락을 채운 값"의 영향이 더 커질 수 있습니다.
즉, 모델이 상품 특성을 배우는 것이 아니라 누락 패턴을 배울 위험이 있습니다.

어디서 확인하나:

- `src/config.py`
- `artifacts/eda/step1/full_data_qc/00_feature_quality/missingness.csv`

### 4. 수집한 전체 상품과 실제 비교에 사용된 상품이 다름

상품 페이지 파일에는 327개 상품이 있지만, 실제 후보 세트에 연결된 상품은 295개만 확인되었습니다.
커버리지는 약 90.2%입니다.

왜 중요한가:
스크래핑한 전체 상품 풀과 실제 학습 및 비교 대상이 완전히 같지 않습니다.
그래서 "327개 전체 상품을 다 반영했다"라고 말하면 부정확합니다.

어디서 확인하나:

- `artifacts/eda/step1/full_data_qc/00_feature_quality/dataset_summary.csv`
- `data/processed/trial_items.csv`

### 5. 두 엔진의 라벨 불일치가 큼

같은 후보 세트라도 `OpenAI`와 `Anthropic`이 고른 순위가 자주 다릅니다.

이미 확인된 수치:

- 1위 상품 일치율: 61.3%
- 1위부터 3위까지 완전히 같은 순위 일치율: 42.3%

왜 중요한가:
두 엔진의 응답을 같은 정답처럼 합쳐서 학습하면 잡음이 적지 않습니다.
한마디로 "정답 자체가 흔들린다"는 뜻입니다.

어디서 확인하나:

- `artifacts/eda/response_diagnostics/overall/engine_agreement_summary.csv`
- `artifacts/eda/response_diagnostics/overall/engine_top1_disagreement_cases.csv`

### 6. 후보를 보여준 순서가 결과에 영향을 줌

후보가 1번, 2번, 3번 자리에 놓였을 때 1위로 뽑히는 비율이 균등하지 않습니다.

이미 확인된 수치:

- `step1` 학습용 분석 기준
  1번 자리 29.5%, 2번 자리 37.9%, 3번 자리 32.6%
- 전체 응답 기준
  1번 자리 29.4%, 2번 자리 37.2%, 3번 자리 34.1%

왜 중요한가:
상품이 정말 좋아서 1위를 받은 것인지, 단지 2번 자리에 있어서 유리했던 것인지 구분해야 합니다.

어디서 확인하나:

- `artifacts/eda/step1/train_only/step1_position_bias.csv`
- `artifacts/eda/response_diagnostics/overall/position_bias_overall.csv`
- `artifacts/eda/response_diagnostics/by_engine/openai/position_bias.csv`
- `artifacts/eda/response_diagnostics/by_engine/anthropic/position_bias.csv`

### 7. 어떤 상품은 많이 나오고 어떤 상품은 적게 나옴

후보로 등장한 횟수가 상품마다 꽤 다릅니다.

이미 확인된 수치:

- 최소 9회
- 가운데값 30회
- 최대 110회

상품 쌍이 같이 등장한 횟수는 대부분 1회지만, 개별 상품 노출량은 고르게 분포하지 않습니다.

왜 중요한가:
많이 등장한 상품은 더 많은 기회를 얻습니다.
그래서 단순한 1위 횟수나 승률을 해석할 때 노출 횟수를 같이 봐야 합니다.

어디서 확인하나:

- `artifacts/eda/step1/full_data_qc/02_candidate_set_quality/item_appearance_counts.csv`
- `artifacts/eda/step1/full_data_qc/02_candidate_set_quality/pair_cooccurrence_counts.csv`
- `artifacts/eda/response_diagnostics/overall/item_exposure_counts.csv`

### 8. 같은 후보 세트가 엔진별로 두 번 존재함

모든 후보 세트는 `OpenAI`용 1행, `Anthropic`용 1행으로 존재합니다.

왜 중요한가:
행 수는 두 배처럼 보이지만, 완전히 독립적인 실험 두 배는 아닙니다.
같은 세 후보에 대한 서로 다른 응답 두 개라고 보는 편이 맞습니다.

어디서 확인하나:

- `response.csv`

### 9. 매핑이 불확실한 상품이 상위 노출 그룹에 있을 수 있음

일부 상품은 정확한 상품 단위가 아니라 브랜드 평균 정보로 연결되어 있습니다.
이런 상품이 많이 등장하면, 특정 상품이 좋아서 상위에 간 것처럼 보여도 실제로는 브랜드 평균 특성의 영향일 수 있습니다.

왜 중요한가:
상품 단위 해석과 브랜드 단위 해석을 섞으면 안 됩니다.

어디서 확인하나:

- `data/processed/trial_items.csv`
- `artifacts/eda/response_diagnostics/overall/item_exposure_counts.csv`

## 응답 데이터 전용 진단은 어디서 보나

응답 자체를 따로 진단하는 스크립트와 결과를 추가해 두었습니다.

- 실행 코드
  `src/response_eda.py`
- 결과 폴더
  `artifacts/eda/response_diagnostics`

이 폴더는 두 부분으로 나뉩니다.

- `overall`
  전체 응답을 합쳐서 본 결과
- `by_engine`
  `OpenAI`와 `Anthropic`을 따로 나눠서 본 결과

### 이 폴더에서 가장 먼저 볼 파일

- `overall/engine_agreement_summary.csv`
  두 엔진이 얼마나 자주 같은 순위를 주는지 요약
- `overall/engine_top1_disagreement_cases.csv`
  1위 상품이 서로 달랐던 실제 사례 모음
- `overall/position_bias_overall.csv`
  후보 제시 순서 효과 요약
- `overall/item_exposure_counts.csv`
  상품별 노출 횟수
- `overall/brand_exposure_counts.csv`
  브랜드별 노출 횟수
- `by_engine/openai/position_bias.csv`
  `OpenAI`만 따로 봤을 때의 순서 효과
- `by_engine/anthropic/position_bias.csv`
  `Anthropic`만 따로 봤을 때의 순서 효과

## 앞으로 EDA를 할 때 추천 순서

### 1단계. 데이터 품질부터 확인

먼저 누락값, 이상치, 중복, 실제 사용 상품 수를 확인합니다.
이 단계는 전체 데이터를 봐도 됩니다.

### 2단계. 후보 세트 구성이 공정한지 확인

특정 상품이 너무 자주 나오지 않는지, 같은 쌍이 자주 묶이지 않는지, 비교 네트워크가 끊겨 있지 않은지 봅니다.

### 3단계. 응답 자체의 특성을 확인

두 엔진이 얼마나 다르게 답하는지, 후보 위치 효과가 있는지, 어떤 엔진이 더 길게 쓰는지 확인합니다.

### 4단계. 그다음에야 학습용 데이터만 가지고 순위 패턴을 해석

가격, 브랜드, 제품 타입, 리뷰 수 같은 정보가 상위 순위와 관련이 있는지 봅니다.
이 단계는 학습용 데이터만 사용해야 합니다.

## 결과를 공유하기 전에 꼭 붙여야 하는 문장

아래 문장은 거의 항상 같이 붙이는 것을 권장합니다.

> 현재 결과는 데이터 수집 품질, 엔진 간 응답 차이, 후보 제시 순서 효과, 상품 노출 불균형의 영향을 받을 수 있으므로 해석에 주의가 필요합니다.

## 실행 방법

전체 상품과 학습용 데이터를 함께 보는 기본 분석:

```bash
python src/eda.py --protocol step1
python src/eda.py --protocol step2
```

응답 자체를 진단하는 분석:

```bash
python src/response_eda.py
```

## 마지막 체크리스트

새로운 분석을 시작하거나 결과를 공유하기 전에 아래 질문에 답해보면 좋습니다.

1. 이 분석은 전체 데이터를 봐도 되는가, 아니면 학습용 데이터만 써야 하는가?
2. 누락이 많은 항목이 결과를 사실상 좌우하고 있지는 않은가?
3. 엔진 두 개를 합친 결과와 엔진별 결과가 크게 다르지 않은가?
4. 후보가 놓인 위치 때문에 유리하거나 불리한 상품이 있지 않은가?
5. 많이 등장한 상품이 단순히 기회가 많아서 상위에 보이는 것은 아닌가?
6. 이 결과를 상품 단위로 말해도 되는가, 아니면 브랜드 단위로만 말해야 하는가?

이 여섯 가지를 먼저 점검하면, EDA 결과를 훨씬 덜 위험하게 해석할 수 있습니다.
