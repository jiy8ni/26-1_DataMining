# 모델 코드 설명

이 폴더에는 모델을 학습하고 결과를 계산하는 코드가 들어 있습니다.  
파일이 많아 보이지만, 각 파일은 하나의 역할만 담당하며 아래 순서대로 연결됩니다.

---

## 전체 흐름

```
설정값 불러오기 (config.py)
       ↓
데이터 불러오고 정리하기 (data.py)
       ↓
모델이 점수 예측하기 (model.py)
       ↓
예측이 틀렸을 때 얼마나 틀렸는지 계산 (loss.py)
       ↓
학습 반복 → 가장 좋은 모델 저장 (train.py)
       ↓
점수를 확률로 변환 (calibration.py)
       ↓
성능 평가 (metrics.py)
       ↓
전체 상품 순위 및 추천 확률 계산 (inference.py)
```

---

## 파일별 설명

---

### config.py — 설정값 모음

모델 학습에 필요한 모든 설정값을 한 곳에 모아둔 파일입니다.  
학습 속도, 모델 구조, 파일 경로 등을 여기서 조정합니다.

주요 설정값:

| 설정 | 기본값 | 의미 |
|------|--------|------|
| 평가 방식 | step1 | step1: 학습에서 본 상품의 새로운 조합 테스트 / step2: 학습에서 한 번도 못 본 새 브랜드 테스트 |
| 신경망 층 크기 | [128, 64, 32] | 신경망의 각 층에 있는 노드 수 |
| 학습률 | 0.001 | 한 번 학습할 때 가중치를 얼마나 바꿀지 |
| 배치 크기 | 64 | 한 번에 몇 개의 비교 세트를 학습할지 |
| 최대 반복 횟수 | 100 | 전체 데이터를 최대 몇 번 반복 학습할지 |
| 조기 종료 기준 | 15 | 검증 성능이 15번 연속으로 나아지지 않으면 학습 중단 |

---

### data.py — 데이터 준비

CSV 파일을 모델이 학습할 수 있는 형태로 변환합니다.

**핵심 역할:**

1. 원본 데이터를 읽어 결측값(비어 있는 셀)을 채웁니다.
2. 수치가 서로 다른 단위(가격은 만 원 단위, 리뷰 수는 개 단위 등)를 동일한 기준으로 맞춥니다. 이렇게 하지 않으면 숫자가 큰 항목이 모델에 지나치게 큰 영향을 미칩니다.
3. 비교 세트(3개 상품 + AI 순위) 단위로 데이터를 묶어 모델에 전달합니다.

**평가 방식 두 가지:**

- **step1**: 학습 / 검증 / 테스트를 비교 세트 단위로 무작위 분할합니다. 같은 상품이 학습과 테스트에 모두 등장할 수 있습니다.
- **step2**: 특정 브랜드를 처음부터 테스트 전용으로 빼둡니다. 모델이 학습 중 한 번도 보지 않은 상품을 얼마나 잘 예측하는지 확인하기 위함입니다.

---

### model.py — 추천 점수 예측 신경망

상품 페이지의 특징 24가지를 입력받아, 그 상품이 AI에게 얼마나 선호될지를 나타내는 **점수 하나**를 출력합니다.

```
입력: 상품 페이지 특징 24개 (가격, 리뷰 수, 텍스트 길이 등)
  ↓  [128개 노드 층]
  ↓  [64개 노드 층]
  ↓  [32개 노드 층]
출력: 추천 점수 1개 (숫자가 클수록 AI에게 더 선호될 가능성 높음)
```

각 층을 통과할 때마다 입력값의 복잡한 패턴을 조금씩 더 추상적으로 압축합니다.  
점수 자체는 확률이 아니며, 상대적인 선호도를 나타내는 숫자입니다.

---

### loss.py — 얼마나 틀렸는지 계산 (Plackett-Luce 손실 함수)

모델이 예측한 점수 순서가 AI가 매긴 순위와 얼마나 다른지를 숫자 하나로 나타냅니다.  
이 숫자(손실값)가 작을수록 모델이 AI의 선호를 잘 학습한 것입니다.

**계산 방식:**

예를 들어 AI가 A > C > B 순위를 줬고, 모델이 아래와 같은 점수를 냈다면:

| 상품 | 모델 점수 |
|------|---------|
| A | 2.0 |
| B | 0.5 |
| C | 1.0 |

1. 먼저 3개 중 A가 1위로 뽑힐 확률을 계산합니다.
2. A를 제외한 나머지 중 C가 2위로 뽑힐 확률을 계산합니다.
3. 두 확률을 곱해 "A > C > B 순서가 나올 전체 확률"을 구합니다.
4. 이 확률이 높을수록 손실값이 낮아집니다.

모델은 손실값을 낮추는 방향으로 반복해서 점수 계산 방식을 조정합니다.

---

### calibration.py — 점수를 확률로 변환

학습이 끝난 모델이 출력하는 점수는 상대적 선호도를 나타내는 숫자입니다.  
이 점수를 "전체 300개 상품 중 이 상품이 선택될 확률"로 바꾸기 위해 보정 과정이 필요합니다.

**온도 보정(Temperature Scaling)이란?**

점수를 확률로 변환할 때 **온도**라는 값으로 나눠서 계산합니다.

- 온도가 낮으면: 1위 상품에 확률이 더 집중됩니다 (모델이 더 확신하는 방향)
- 온도가 높으면: 전체 상품에 확률이 더 고르게 분산됩니다 (모델이 덜 확신하는 방향)

최적의 온도값은 학습 데이터가 아닌 **검증 데이터**에서 가장 정확한 확률을 내는 값을 찾아 결정합니다.  
모델을 다시 학습하는 것이 아니라, 이미 학습된 점수를 확률로 변환하는 방식만 조정하는 과정입니다.

---

### metrics.py — 성능 평가 지표

학습된 모델이 얼마나 잘 예측하는지를 여러 각도에서 측정합니다.

**순위 정확도 지표:**

| 지표 | 의미 |
|------|------|
| 1위 정확도 | AI가 1위로 고른 상품을 모델도 1위로 예측한 비율 |
| 쌍별 정확도 | 두 상품을 비교했을 때 선호 방향을 맞힌 비율 (3개 중 2개씩 비교하면 3쌍 생성) |
| NDCG@3 | 1위를 맞히는 것에 더 큰 점수를 주는 가중 정확도 (1위를 틀리면 큰 감점) |
| 켄달 타우 | 전체 순위 순서가 AI 순위와 얼마나 일치하는지 (-1에서 1 사이, 1이 완벽 일치) |

**확률 보정 지표:**

| 지표 | 의미 |
|------|------|
| 음의 로그 우도 | 모델이 AI가 실제로 선택한 순위에 얼마나 높은 확률을 부여했는지 (낮을수록 좋음) |
| 브라이어 점수 | 예측 확률과 실제 결과가 얼마나 가까운지 (낮을수록 좋음) |

---

### inference.py — 전체 상품 순위 계산

학습이 끝난 모델을 사용해 전체 300개 상품에 점수를 매기고, 순위와 추천 확률을 계산합니다.

**출력 결과:**

| 항목 | 의미 |
|------|------|
| 추천 점수 | 모델이 계산한 원시 점수 |
| 전체 순위 | 300개 중 몇 위인지 (1위가 가장 추천될 가능성 높음) |
| 상위 퍼센트 | 전체 상품 중 상위 몇 퍼센트에 속하는지 |
| 추천 확률 | 300개 상품 중 이 상품이 선택될 확률 |
| 평균 대비 배율 | 평균적인 상품보다 몇 배 추천될 가능성이 높은지 |

---

### train.py — 학습 실행

위 모든 파일을 연결해 실제 학습을 진행하는 파일입니다.  
`python train.py`를 실행하면 아래 순서로 진행됩니다.

```
1. 설정값과 데이터 불러오기
2. 모델 초기화
3. 학습 반복
   - 비교 세트 64개씩 묶어서 점수 예측
   - 손실값 계산 (예측 순위와 AI 순위 비교)
   - 손실값을 줄이는 방향으로 모델 조정
   - 검증 데이터로 중간 성능 확인
   - 성능이 15번 연속 나아지지 않으면 조기 종료
4. 가장 좋은 모델 저장
5. 온도 보정값 결정
6. 테스트 데이터로 최종 성능 측정 및 출력
```

학습 진행 상황(손실값 변화, 학습률 변화)은 Weights & Biases 대시보드에서 실시간으로 확인할 수 있습니다.

## 입력 특징 목록 (24가지)

모델이 각 상품 페이지에서 읽어들이는 수치 목록입니다.

| 분류 | 항목 |
|------|------|
| 텍스트 | 전체 텍스트 길이, 단락 수, 섹션 수, 목록 항목 수, 표 수 |
| 신뢰도 | 구체적인 숫자 표현 수, 외부 기관 인용 수, 모호한 표현 수, 수치 구체성 비율 |
| 가격/용량 | 가격, 용량(ml) |
| 성분/효능 | 주요 성분 수, pH 언급 여부, 효능 키워드 수, 텍스처 키워드 수 |
| 피부 타입 | 대상 피부 타입 수 |
| 리뷰 | 평점, 리뷰 수 |
| 인증/신뢰 | 인증 마크 수, 부정적 표현 수 |
| 페이지 구성 | 구조화 데이터 포함 여부 및 항목 수 |
| 소셜 증거 | 소셜 증거 요소 수 |
| 전문가 점수 | AI 친화도 종합 점수 |
